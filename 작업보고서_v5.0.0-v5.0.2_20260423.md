## 2. 배포 중 발견된 버그 수정

### 2-1. Docker 컨테이너 부팅 실패
- **증상:** gunicorn 워커가 DB 파일을 열지 못해 부팅 실패
- **원인:** SQLite 볼륨 마운트가 파일 단위로 되어 있어 WAL/SHM 파일 생성 불가
- **해결:** 파일 마운트 → 디렉토리 마운트로 변경 (`-v /root/logic-analysis-deploy/data:/app/data`)

### 2-2. TypeError: k.volume.replace is not a function
- **증상:** 분석 결과 화면에서 콘솔 에러 발생, 일부 데이터 미표시
- **원인:** `k.volume` 값이 숫자형(number)인 경우 `.replace()` 메서드 호출 불가
- **해결:** `String(k.volume || '0').replace(...)` 으로 명시적 문자열 변환

### 2-3. 성별 50/50, 연령대 0% 표시
- **증상:** 데이터랩 인구통계 섹터에서 성별이 항상 50/50, 연령대가 0%로 표시
- **원인:** API 응답의 `group` 필드가 `results` 레벨이 아닌 `data` 포인트 내부에 위치하는데, 파싱 로직이 잘못된 레벨에서 조회
- **해결:** `data_points` 배열 내부에서 `group` 값을 정확히 추출하도록 파싱 로직 수정

### 2-4. CSS 캐시 버전 불일치
- **증상:** 스타일 업데이트가 브라우저에 반영되지 않음
- **원인:** `index.html`의 CSS 파일 버전이 `v4.4.0`으로 남아있어 브라우저 캐시 사용
- **해결:** `styles.css?v=4.4.0` → `styles.css?v=5.0.0` 으로 업데이트

### 2-5. top5 변수 오류
- **증상:** top5 상품이 10개 표시
- **원인:** `page1_products[:10]` 으로 잘못 설정
- **해결:** `page1_products[:5]` 로 수정

## 3. 코드 최적화 및 리팩토링 (v5.0.1)

전체 코드를 개발자/사용자 관점에서 점검하고 5가지 이슈를 발견 후 수정했습니다.

### 3-1. 데이터랩 성장률 API 병렬화
- **이전:** 6개 API를 순차 호출 (약 6~12초 소요)
- **이후:** `ThreadPoolExecutor`로 병렬 호출 (약 2~3초로 단축)
- **파일:** `backend/datalab.py`

### 3-2. 리포트 내보내기 N+1 쿼리 제거
- **이전:** 리포트 내보내기 시 상품 수만큼 DB 쿼리 반복 호출
- **이후:** 이미 메모리에 있는 데이터 사용으로 불필요한 DB 호출 제거
- **파일:** `backend/main.py`

### 3-3. 하드코딩된 superadmin 사용자명 수정
- **이전:** `currentUser.username === 'yoosub92'` 로 하드코딩
- **이후:** `currentUser.role === 'superadmin'` 으로 역할 기반 체크
- **파일:** `frontend/js/components/App.jsx`, `backend/main.py`

### 3-4. 데이터랩 로딩 인디케이터 추가
- **이전:** 데이터랩 분석 중 화면에 아무 피드백 없음
- **이후:** 스피너 + "데이터랩 분석 중..." 텍스트 표시
- **파일:** `frontend/js/components/App.jsx`, `frontend/css/styles.css`

### 3-5. 데이터랩 섹터 반응형 그리드 적용
- **이전:** 인라인 스타일로 고정 그리드 → 모바일에서 깨짐
- **이후:** CSS 클래스(`card-grid-2/3/4`) 적용으로 반응형 대응
- **파일:** 5개 데이터랩 컴포넌트 (Season, Growth, Demographics, CategoryKeywords, Trend)

## 4. 성능 개선 (v5.0.2)

### 4-1. gunicorn 워커 수 증가
- **이전:** 3개 워커
- **이후:** 6개 워커
- **효과:** 동시 요청 처리 용량 2배 증가 (약 30~36명 동시 사용 가능)
- **파일:** `Dockerfile`

### 4-2. 데이터랩 API 결과 메모리 캐싱
- **구현:** 같은 키워드+카테고리 조합으로 1시간 내 재분석 시 API 호출 없이 즉시 반환
- **사양:** TTL 1시간, 최대 200건, 스레드 안전(threading.Lock), 자동 만료 정리
- **효과:** 반복 분석 시 데이터랩 섹터 즉시 로딩, 네이버 API 호출 횟수 대폭 감소
- **파일:** `backend/datalab.py`

## 5. 배포 중 추가 이슈 해결

### 5-1. VPS 디스크 100% 풀
- **증상:** Docker 컨테이너 로그 출력 불가, 서비스 접속 시 "네트워크 오류"
- **원인:** Docker 미사용 이미지/캐시 누적으로 디스크 35GB 중 100% 사용
- **해결:** `docker system prune -a`, `docker image prune -a` 실행 → 4.5GB 확보 (74%로 감소)

### 5-2. 캐시 키 dict 비교 오류
- **증상:** 데이터랩 섹터 전부 미표시
- **원인:** `related_keywords`가 dict 리스트인데 `sorted()`로 비교 시 `'<' not supported between instances of 'dict' and 'dict'` 에러
- **해결:** dict에서 keyword 문자열만 추출 후 정렬하도록 수정

### 5-3. superadmin role 미설정
- **증상:** 설정 탭 미표시
- **원인:** DB에서 yoosub92 계정의 role이 `admin`으로 되어 있었음 (코드는 `superadmin` 체크로 변경됨)
- **해결:** DB에서 role을 `superadmin`으로 업데이트
